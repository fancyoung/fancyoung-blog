CSS实现背景透明完美解决方案

翻译自: CSS Background Transparency Without Affecting Child Elements, Through RGBa And Filters

当今的大部分网页设计都包含了半透明的元素.但用CSS实现想要的效果却没那么简单.

现存的问题

如果我们想要一个半透明背景,有两种实现方式:

  • 利用CSS和opacity属性
  • 建立一个24位PNG背景图片
    用opacity的问题除了
    需要通过麻烦的语法来兼容所有浏览器,
    还有不单该元素自身背景透明了,它的子元素也会透明.
    所以里面所有的文字也是透明的,这一般不是想要的效果.
    你可以做一些CSS trick,通过建立额外的元素来解决这个问题,但这种解决方案很恶心.

用PNG的问题是会额外增加HTTP请求,
而且图片比几行css代码要大得多-
尤其考虑到图片不能太小以避免
IE下24位PNG图透明时引起的内存泄漏.

继续

不幸的是IE不支持RGBa(IE678都不行).
幸运的是微软在2000年时疯狂的给IE加上了各种过滤器.
其中有一个是渐变过滤器(the gradient filter).aspx),
我们只需要把开始结束都设置成一个颜色就可以了.
你现在会想,”那我们怎么将其透明呢?”.
你只需要用16进制值来定义颜色就可以了.
下面是用渐变过滤器实现和上面同等透明效果的代码:

你可以看到定义里的8位16进制数字,
前两位代表透明度,接下来两位是红色,接着就是绿色和蓝色.
注意两位16进制颜色的范围是0~255(ff).
那我们怎么将0.6转化为16进制数呢?

这里就需要用到一些数学知识了.
基本上,我们只需要将需要的透明度(比如0.6)乘以255,再转化为16进制即可.
还有一个简单的办法是利用
Google’s Search Features,
你只需要在Google里搜索
0.6 * 255 in hex
就能得到答案.
不过Google计算器仿佛只能处理整数,而
0.3 * 255 in hex
它便无法得出结果.

还有一种更快的方法就是利用JavaScript,
